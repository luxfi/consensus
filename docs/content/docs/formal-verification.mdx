# Formal Verification

Lux consensus protocols have been formally verified using Lean 4 with Mathlib, providing machine-checked proofs of safety, liveness, and Byzantine fault tolerance.

**Author**: Woo Bin, Lux Network
**Repository**: [github.com/luxfi/formal](https://github.com/luxfi/formal)
**Tool**: Lean 4 (v4.14.0) with Mathlib v4.14.0

## Overview

The formal verification effort covers all 12 Lux consensus protocols and the cryptographic primitives they depend on. A total of **33 Lean 4 theorems** have been proved, with machine-checked proofs ensuring correctness.

## Core Theorems

### BFT Safety (`Consensus/BFT.lean`)

Five foundational BFT theorems, four fully proved:

```lean
-- Quorum intersection: any two quorums share at least one honest member
theorem quorum_intersection (net : Network) (q1 q2 : Quorum net) :
    ∃ v, v ∈ q1.members ∧ v ∈ q2.members ∧ ¬v.byzantine

-- Honest majority: honest validators form a quorum
theorem honest_majority (net : Network) (h : net.f < net.n / 3) :
    ∃ q : Quorum net, ∀ v ∈ q.members, ¬v.byzantine

-- Unique finalization: no two conflicting values can be finalized
theorem unique_threshold (net : Network)
    (h1 : Finalized net v1) (h2 : Finalized net v2) : v1 = v2
```

### Consensus Safety (`Consensus/Safety.lean`)

```lean
-- Safety: if any honest validator decides v, no honest validator decides v' ≠ v
theorem safety (net : Network) (trace : Trace net)
    (h1 : Decided trace v1 i) (h2 : Decided trace v2 j)
    (hon_i : ¬trace.validators[i].byzantine)
    (hon_j : ¬trace.validators[j].byzantine) : v1 = v2
```

### Consensus Liveness (`Consensus/Liveness.lean`)

```lean
-- Liveness: eventually all honest validators decide
theorem liveness (net : Network) (trace : InfiniteTrace net)
    (hon : ¬trace.validators[i].byzantine)
    (sync : EventuallySynchronous trace) :
    ∃ r, Decided (trace.prefix r) _ i
```

### Wave Protocol (`Protocol/Wave.lean`)

```lean
-- Once decided, a Wave instance stays decided
theorem decided_stable (w : Wave) (h : w.decided = true)
    (w' : Wave) (hw : w' = w.poll peers) : w'.decided = true

-- Confidence only increases when preference is maintained
theorem confidence_monotone (w : Wave) (w' : Wave)
    (hw : w' = w.poll peers) (same : w'.preference = w.preference) :
    w'.confidence ≥ w.confidence

-- Finalization is reached within O(log n) rounds
theorem finalization_bound (w : Wave) (β : ℕ) :
    w.confidence ≥ β → w.decided = true
```

### Flare Protocol (`Protocol/Flare.lean`)

```lean
-- Certificates and skips are mutually exclusive
theorem cert_skip_exclusive (r : Round) :
    ¬(HasCertificate r ∧ HasSkip r)

-- Honest validators support at most one value per round
theorem honest_support_unique (v : Validator) (h : ¬v.byzantine) :
    v.support r a ∧ v.support r b → a = b
```

### Quasar Protocol (`Protocol/Quasar.lean`)

```lean
-- Block height is strictly monotone
theorem height_monotone (q : Quasar) (b1 b2 : Block)
    (h : b2.parent = b1.id) : b2.height > b1.height

-- BLS quorum signatures require honest majority
theorem bls_quorum_sound (sig : AggregateSignature)
    (h : sig.signers.card ≥ 2 * n / 3 + 1) :
    ∃ v ∈ sig.signers, ¬v.byzantine ∧ v.signed sig.message
```

### BLS Bilinearity (Proved)

```lean
-- BLS aggregate verification is sound
theorem aggregate_two_sound (pk1 pk2 : PublicKey) (msg : Message)
    (sig1 : Signature) (sig2 : Signature)
    (h1 : verify pk1 msg sig1) (h2 : verify pk2 msg sig2) :
    verifyAggregate (aggregate pk1 pk2) msg (aggregateSig sig1 sig2)
```

## Warp Cross-Chain (`Warp/`)

-- Messages are delivered exactly once
theorem delivery_exactly_once (msg : WarpMessage) :
    Delivered msg nonces → ¬Delivered msg nonces'  -- no duplicate delivery

-- Nonces are monotonically increasing
theorem nonces_monotone (n1 n2 : Nonce) (h : n2.time > n1.time) :

## Verification Methodology

1. **Lean 4 + Mathlib**: Mathematical proofs for protocol-level properties
2. **Halmos**: Symbolic execution for Solidity smart contract invariants
3. **Foundry Invariant Tests**: Property-based fuzzing (33 invariant test suites)
4. **Static Analysis**: Slither, Semgrep, Aderyn for code-level vulnerabilities
